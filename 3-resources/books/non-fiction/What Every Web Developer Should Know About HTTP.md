# What Every Web Developer Should Know About HTTP
by [[K. Scott Allen]]

## Highlights
> When a website wants to give a user a cookie, it uses a Set-Cookie header in an HTTP response.

> The HttpOnly flag tells the browser to prevent access to the cookie from JavaScript. The cookie exists only to travel out in the header of every HTTP request message. Browsers that implement HttpOnly will not allow JavaScript to read or write the cookie on the client.

> Most web browsers will allow you to disable third-party cookies, but third-party cookies are on by default.

> HTTPS is computationally expensive, and large sites often use specialized hardware (SSL accelerators) to take the cryptographic computation load off the web servers. HTTPS traffic is also impossible to cache in a public cache, however, user agents might keep HTTPS responses in their private cache. Finally, HTTPS connections are expensive to set up and require an additional handshake between the client and server to exchange cryptographic keys and ensure everyone is communicating with the proper secure protocol. Persistent connections can help to amortize this cost.
